# Internationalization (i18n)

Bible Reader supports **English (`en`)** and **German (`de`)** for the application UI.

## How locale is chosen

1. **Logged-in users:** `users.locale` (Settings → Reading preferences).
2. **Guests:** session key `:locale` (set via `/locale/:locale` or browser detection).
3. **First visit:** `Accept-Language` header (`de*` → `de`, otherwise `en`).
4. **Fallback:** `en`.

The `BibleReaderWeb.Plugs.Locale` plug runs on every HTML request and sets `Gettext` + `conn.assigns.locale`. LiveViews use `on_mount {BibleReaderWeb.Locale, :set_locale}`.

## Bible text vs UI language

Imported scripture text (e.g. Elberfelder via `mix scripture.import deuelbbk`) is **independent** of UI locale. The default translation is shown regardless of `en`/`de` UI setting.

## Book names

Display names use the Gettext domain **`books`** with OSIS codes as msgids (`GEN`, `MAT`, …). English names remain in the database; UI uses `Scripture.book_display_name/2`.

## Adding or updating strings

1. Wrap user-visible text in `gettext/1`, `ngettext/3`, or `Gettext.dgettext/3` (for dynamic domains).
2. Run:

   ```bash
   mix gettext.extract --merge
   ```

3. Update `priv/gettext/de/LC_MESSAGES/default.po` (and `errors.po` for validation messages).
4. Regenerate German defaults (optional helper):

   ```bash
   mix run priv/gettext/scripts/generate_de.exs
   ```

   Edit the script’s translation maps when adding many new strings.

## Relative times

`ReadingPlan.RelativeTime` returns structured values (`:today`, `{:days, n}`, …). Format for display with `BibleReaderWeb.RelativeTimeFormat.format/2`.

## Calendar dates

Month and weekday names use `BibleReader.I18n.CalendarFormat` (`format_long/2`, `format_datetime/2`) because Elixir's `Calendar.strftime/2` always emits English `%A` / `%B` / `%b` names. Effective-date banners, picker labels, and reading-history timestamps go through this module.

## Emails

`Accounts.UserNotifier` sends messages in the recipient’s `user.locale` via `Gettext.with_locale/2`.
